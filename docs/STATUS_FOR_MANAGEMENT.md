### Что блокирует production-уровень

Текущий статус модели — `provisional_not_production`. Это **сознательная остановка**, не недоработка. Причина — качество входных данных от Unreal-плагина в батче `0003`:

- из 24005 объектных слотов плагин записал 10180 (42%) как all-zero placeholder для невидимых колёс
- ещё 10505 (44%) — partial-zero или out-of-bounds
- usable yield: 1722 колеса (7.17%)

Это **проблема стороны Unreal-плагина**, не ML. Подробный разбор и список фиксов: `docs/UNREAL_EXPORT_0003_FEEDBACK.md`.

### Что нужно для production-релиза

Единственный блокирующий шаг: **получить чистый батч 0004 от Unreal-разработчика (Игорь)** с двумя фиксами:

1. Не писать `.txt` для невидимых/offscreen колёс (вместо all-zero placeholder).
2. Экспортировать `bbox_xyxy` около полного силуэта колеса (сейчас bbox строим эвристикой из 4 sphere-точек, что слабее).

Когда придёт 0004:

| День | Действие | Кто |
| --- | --- | --- |
| 0 | Получаем 0004 | Igor → Eduard |
| +0.5 | Прогоняем acceptance pipeline, проверяем quality gate | автоматически |
| +1 | Перетренировка на 0004 (MPS, ~30 мин) + parity TFLite | автоматически |
| +1 | Обновляем `MOBILENETV2_LITERT_HANDOFF.md`, отправляем AR | автоматически |

То есть **с момента получения чистого 0004 до production handoff — один рабочий день**.

### Что AR-команда может делать прямо сейчас

Они **не заблокированы** ML-стороной. Provisional TFLite уже отдан, схема финальная, AR может:

- интегрировать TFLite в Android-runtime,
- проверить препроцессинг (RGB→NCHW float32, 640×640, normalize 0–1),
- проверить декодирование (stride 32, output grid 20×20),
- проверить AR JSON парсинг,
- начать raycast + RANSAC + накопление по K кадров,
- проверить latency на реальном устройстве.

Замена provisional → production произойдёт без изменения контракта или схемы — AR просто подгрузит новый `.tflite`.

### Risk register

- **Главный риск:** Igor не успевает выдать 0004 к сроку → ML может натренировать на синтетике из 0001+0002 как fallback, но качество будет хуже provisional на 0003.
- **Средний риск:** на 0004 окажутся другие проблемы геометрии → одна итерация feedback, +1 день.
- **Низкий риск:** TFLite latency на устройстве AR не устроит → есть план перехода на INT8-квантизацию (документирован в handoff).
